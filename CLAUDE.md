# CLAUDE.md — H&M Services (Fairhope)

## Source of truth

**This repository (`EliteProdigy1/Hmservicesfairhope`) is the single source of
truth for the live H&M Services website.**

- The site is served from the **repository root** (`index.html`, `assets/`,
  `core/`, `netlify.toml` with `publish = "."`).
- Netlify deploys from the **`main`** branch.
- All H&M changes — hero, copy, forms, assets, config — are made **here** and
  deployed from `main`.

## Do not sync with the factory

This site was originally generated by the EP Website Factory in
`EliteProdigy1/EP-Media-Agency` (`config/hm-services.json`,
`assets/hm-services/`, `dist/hm-services/`). That factory copy is now
**frozen/legacy** for H&M:

- **Do NOT** run `ep:build` for `hm-services` and push the output over this
  live site.
- **Do NOT** back-port changes made here into EP-Media-Agency, and do NOT pull
  factory changes into this repo.
- Treat any divergence between this repo and the factory copy as expected —
  this repo wins.

## Contact form

The estimate form is a Netlify Form named **`hm-services-contact`**
(`data-netlify="true"`, honeypot `bot-field`). After a deploy with form
detection enabled, add an email notification to **eliteprodigyway@gmail.com**
in the Netlify dashboard (Forms → hm-services-contact → notifications).

## Hero image

Hero background lives at `assets/hero-768.webp`, `assets/hero-1200.webp`, and
`assets/hero-1600.webp`, served responsively via `srcset` / `sizes="100vw"`
in `index.html`. `og:image` / `twitter:image` point to `hero-1600.webp`.

## Cache-busting when replacing an image in place

`netlify.toml` caches `/assets/*` for 30 days (`max-age=2592000`). If you
replace a photo but keep the **same filename**, returning visitors keep seeing
the old cached image. So whenever you overwrite an existing asset (e.g.
`gallery-01-800.webp`), **bump the `?v=N` query string** on every reference to
it in `index.html` (service cards + the `HM_DATA.gallery` array). Brand-new
filenames don't need it. Current version token: `?v=2`.
